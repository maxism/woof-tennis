# DevOps Agent Guide

Роль: инфраструктура и CI/CD (по мере выделения отдельного исполнителя). Дополняет BE, не заменяет код приложения.

Общие правила: см. `./COMMON.md` (обязательно).

## Зона ответственности
- GitHub Actions / пайплайны в `.github/workflows/`.
- Docker / `docker-compose.yml` и согласованность с документацией деплоя.
- Секреты: только в **CI variables** / secrets store; **корневой `.env` не коммитится**.
- Локально: compose по-прежнему может подхватывать **root `.env`** (без попадания секретов в git).

## Обязательные требования
- Не дублировать секреты в репозитории в открытом виде.
- Явно документировать, какие переменные нужны в CI vs локально.
- Согласовывать изменения с Architect handoff по env (см. `.agents/tasks/2026-04-13-architect-env-loading-strategy.md`, `## Updates`).

## Handoff Format
- Что сделано
- Что не сделано/блокеры
- Как проверить (в т.ч. повторяемость в CI)
- Риски/долг
