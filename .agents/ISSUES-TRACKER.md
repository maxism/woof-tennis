# Issues Tracker Standard

Единый учет открытых и закрытых проблем проекта.

## Где вести учет
- Основной реестр: `.agents/issues/ISSUES.md`
- Новые записи добавляются только через PM Coordinator.

## Статусы
- `OPEN` - проблема подтверждена, работа не начата.
- `IN_PROGRESS` - назначен исполнитель, идет работа.
- `BLOCKED` - есть внешний блокер.
- `CLOSED` - проблема решена и проверена.

## Приоритеты
- `P0` - критично для релиза/продакшна.
- `P1` - важно, но есть временный обход.
- `P2` - улучшение/неблокирующий дефект.

## Формат записи
```md
| ID | Title | Priority | Status | Owner | Created | Updated | Source Task | Notes |
|----|-------|----------|--------|-------|---------|---------|------------|-------|
| WT-001 | ... | P0 | OPEN | BE | 2026-04-13 | 2026-04-13 | .agents/tasks/...md | ... |
```

## Правила обновления
- При выдаче новой задачи с исправлением дефекта PM добавляет/обновляет запись в реестре.
- При переводе в `CLOSED` обязательно указать:
  - ссылку на выполненную задачу;
  - кто валидировал закрытие (обычно QA или PM);
  - короткий итог проверки.
- Периодически архивировать закрытые элементы по правилам `.agents/BACKLOG-GROOMING.md`.
