# WoofTennis Agent Common Contract

Единый операционный контракт для всех проектных агентов.

## 1) Продуктовый фокус
- Продукт: Telegram Mini App для букинга теннисных тренировок.
- MVP приоритет: авторизация, профили, локации, слоты, бронирования, отзывы, нотификации.
- UI/тексты для пользователей: русский как основной язык.
- Базовые принципы: mobile-first, минималистичный интерфейс, светлая и темная темы.

## 2) Рабочая среда
- Монорепо: `apps/web`, `apps/api`, `packages/shared`.
- Любые изменения API-контракта синхронизируются через `@wooftennis/shared`.
- Перед задачей проверь релевантные документы в `docs/`.
- Не менять чужие несвязанные изменения в git-дереве.

## 3) Definition of Done (для каждой задачи)
- Реализованы требования задачи без изменения несвязанной логики.
- Пройдены локальные проверки для затронутого скоупа.
- Обновлены типы/контракты в `packages/shared`, если менялся формат данных.
- Добавлены/обновлены тесты для критичной логики.
- Подготовлен короткий handoff: что сделано, что проверить, риски.

## 4) Чеклист качества перед handoff
- Типы и импорты не сломаны.
- Ошибки и пользовательские сообщения на русском (если user-facing).
- Нет хардкода строк интерфейса там, где уже применяется i18n.
- Изменения минимальны и атомарны.

## 5) Команды проверки
- Общие: `npm run lint`, `npm run build`, `npm run test`
- FE: `npm run lint --workspace=@wooftennis/web`, `npm run build --workspace=@wooftennis/web`
- BE: `npm run lint --workspace=@wooftennis/api`, `npm run test --workspace=@wooftennis/api`

## 6) Формат ответа от любого агента
- Что сделано (кратко, по пунктам).
- Что не успел/блокеры.
- Как проверить (команды + ручные шаги).
- Риски/долг, если остались.
