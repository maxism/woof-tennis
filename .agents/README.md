# Agent Playbook for WoofTennis

В этой папке находятся инструкции для рабочих ролей проекта:

- `COMMON.md` — общий контракт для всех
- `PM-COORDINATOR.md` — project/product manager-координатор
- `FE-DEV.md` — фронтенд агент
- `BE-DEV.md` — бэкенд агент
- `ARCHITECT.md` — архитектор
- `DESIGNER.md` — дизайнер
- `QA.md` — QA
- `DEVOPS-DEV.md` — DevOps / CI/CD (опционально, если роль выделена)

## Как использовать в Cursor

Рабочий режим: задачи ставятся PM напрямую в чате, без обязательного ведения task-файлов.

Базовые требования к любому агенту:
1) Прочитать `.agents/COMMON.md`
2) Прочитать профиль роли `.agents/<ROLE>.md`
3) Выполнить задачу в рамках роли и вернуть handoff в чат
