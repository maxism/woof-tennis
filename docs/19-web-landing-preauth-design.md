# WoofTennis — Дизайн pre-auth лендинга (web)

## Цель экрана

Лендинг до авторизации веб-сайта должен:
- быстро объяснять ценность продукта;
- снижать тревожность перед входом;
- вести в один главный сценарий: `Войти через Telegram`.

## UX-принцип

- Один экран, одна основная цель, один главный CTA.
- Friendly тон: просто, без технической терминологии.
- RU primary, EN only as helper microcopy.

## Информационная архитектура экрана

1. **Hero**
   - Лого + имя продукта.
   - Заголовок: `Теннис проще. Тренировки и игры в Telegram.`
   - Подзаголовок (короткий и теплый): `Назначайте тренировки и игры без лишних чатов и путаницы.`
   - Primary CTA: `Войти через Telegram`.
   - Secondary CTA: `Как это работает`.

2. **How it works (3 шага)**
   - `Создай событие`
   - `Пригласи игрока`
   - `Играй без путаницы`

3. **Ролевые преимущества**
   - Карточка `Для игрока`: вижу свои события, локации, статусы.
   - Карточка `Для тренера`: создаю слот, назначаю игрока, отправляю invite.

4. **Блок доверия**
   - Заголовок: `Безопасный вход через Telegram`.
   - Текст: данные входа проверяются сервером, вход в один клик.

5. **Footer**
   - `Политика`
   - `Поддержка`

## Контент и копирайт (RU primary)

### Основные тексты
- H1: `Теннис проще. Тренировки и игры в Telegram.`
- Subtitle: `Управляйте событиями в одном месте: создавайте, приглашайте, подтверждайте.`
- CTA primary: `Войти через Telegram`
- CTA secondary: `Как это работает`

### Microcopy рядом с кнопкой входа
- `Нажимая кнопку, вы входите через официальный Telegram Login Widget.`

### Empty-state fallback
- `Не удалось загрузить виджет. Обновите страницу или попробуйте позже.`

## Визуальные правила

- Стиль: минимализм, мягкие отступы, крупные радиусы, без тяжелых теней.
- Светлая/темная тема обязательны.
- Акцент: `#10B981`.
- Ширина контейнера: mobile-first, max-width `720px` для текста.
- В hero только один яркий элемент: primary CTA.

## Состояния экрана

- `default`: весь лендинг + доступный Login Widget.
- `loading`: skeleton для блока виджета.
- `widget_error`: fallback сообщение и кнопка `Обновить`.
- `auth_in_progress`: disabled CTA + короткий статус `Входим...`.

## Навигационное поведение

- До авторизации показывается только pre-auth landing.
- После успешного логина редирект на `Главная` (`/`).
- Если пользователь уже авторизован — лендинг пропускается.

## Что проверить в QA

- Понимание ценности экрана за 5 секунд (без скролла).
- Видимость главного CTA на первом экране в light/dark.
- Корректная работа в mobile webview и desktop browser.
- Длина русских строк не ломает layout.
